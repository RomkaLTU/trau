# Issue tracker: Linear

Issues and PRDs for this repo live in the Codesomelabs Linear workspace:

- Team: `COD` (`Codesomelabs`)
- Project: `trau`

Use the connected Linear tools for all operations.

## Conventions

- Create issues in team `COD` and project `trau`.
- Read an issue with its full description, comments, and dependency relations before changing it.
- Publish AFK-ready implementation issues with the `ready-for-agent` label.
- Express dependencies with Linear's blocking relations and repeat the blocker in the issue body's `Blocked by` section.
- Use `CONTEXT.md` vocabulary in titles and descriptions: Provider, Model, Route, Fallback provider, and Provider override.
- Do not modify or close a parent issue when publishing child or related implementation issues unless the user explicitly asks.

## When a skill says "publish to the issue tracker"

Create a Linear issue in team `COD`, project `trau`, using the requested triage label and dependency relations.

## When a skill says "fetch the relevant ticket"

Fetch the Linear issue by its `COD-` identifier, including comments and relations.
